Apa itu java ? Sebelum kita membahas terlalu mendalam mengenai real code dalam java ada baiknya terlebih dahulu kita mengenal latar belakang, pengertian dan pemahaman dasar java, itu akan membantu kita dikemudian hari, jangan pernah MALU untuk memulai dari hal yang terkecil karena faktanya kita adalah makhluk kecil di periode lalu yang kini berjalan bersama waktu.

لسَّلاَمُ عَلَيْكُمْ وَرَحْمَةُ اللهِ وَبَرَكَاتُهُ

<p><b>Java</b> adalah <a title="Bahasa pemrograman" href="http://id.wikipedia.org/wiki/Bahasa_pemrograman">bahasa pemrograman</a> yang dapat dijalankan di berbagai <a title="Komputer" href="http://id.wikipedia.org/wiki/Komputer">komputer</a> termasuk <a title="Telepon genggam" href="http://id.wikipedia.org/wiki/Telepon_genggam">telepon genggam</a>. Bahasa ini awalnya dibuat oleh <a title="James Gosling" href="http://id.wikipedia.org/wiki/James_Gosling">James Gosling</a> saat masih bergabung di <a title="Sun Microsystems" href="http://id.wikipedia.org/wiki/Sun_Microsystems">Sun Microsystems</a> saat ini merupakan bagian dari <a class="mw-disambig" title="Oracle" href="http://id.wikipedia.org/wiki/Oracle">Oracle</a> dan dirilis tahun <a title="1995" href="http://id.wikipedia.org/wiki/1995">1995</a>. Bahasa ini banyak mengadopsi sintaksis yang terdapat pada <a title="C (bahasa pemrograman)" href="http://id.wikipedia.org/wiki/C_(bahasa_pemrograman)">C</a> dan <a title="C++" href="http://id.wikipedia.org/wiki/C%2B%2B">C++</a>namun dengan sintaksis model objek yang lebih sederhana serta dukungan rutin-rutin aras bawah yang minimal. Aplikasi-aplikasi berbasis java umumnya dikompilasi ke dalam <a class="mw-redirect" title="P-code" href="http://id.wikipedia.org/wiki/P-code">p-code</a> (<i>bytecode</i>) dan dapat dijalankan pada berbagai <a class="mw-redirect" title="Mesin virtual java" href="http://id.wikipedia.org/wiki/Mesin_virtual_java">Mesin Virtual Java (JVM)</a>. Java merupakan bahasa pemrograman yang bersifat umum/non-spesifik (<i>general purpose</i>), dan secara khusus didisain untuk memanfaatkan dependensi implementasi seminimal mungkin. Karena fungsionalitasnya yang memungkinkan aplikasi java mampu berjalan di beberapa platform <a title="Sistem operasi" href="http://id.wikipedia.org/wiki/Sistem_operasi">sistem operasi</a>yang berbeda, java dikenal pula dengan slogannya, &#8220;<i>Tulis sekali, jalankan di mana pun</i>&#8220;. Saat ini java merupakan bahasa pemrograman yang paling populer digunakan, dan secara luas dimanfaatkan dalam pengembangan berbagai jenis perangkat lunak aplikasi ataupun aplikasi berbasis web.</p>
<h2><span id="Sejarah_perkembangan" class="mw-headline">Sejarah perkembangan</span></h2>
<p><a title="Bahasa pemrograman" href="http://id.wikipedia.org/wiki/Bahasa_pemrograman">Bahasa pemrograman</a> Java terlahir dari <a class="new" title="The Green Project (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=The_Green_Project&amp;action=edit&amp;redlink=1">The Green Project</a>, yang berjalan selama 18 bulan, dari awal tahun <a title="1991" href="http://id.wikipedia.org/wiki/1991">1991</a> hingga musim panas <a title="1992" href="http://id.wikipedia.org/wiki/1992">1992</a>. Proyek tersebut belum menggunakan versi yang dinamakan <a class="mw-redirect" title="Oak" href="http://id.wikipedia.org/wiki/Oak">Oak</a>. Proyek ini dimotori oleh Patrick Naughton, Mike Sheridan, dan <a title="James Gosling" href="http://id.wikipedia.org/wiki/James_Gosling">James Gosling</a>, beserta sembilan pemrogram lainnya dari<a title="Sun Microsystems" href="http://id.wikipedia.org/wiki/Sun_Microsystems">Sun Microsystems</a>. Salah satu hasil proyek ini adalah maskot <i>Duke</i> yang dibuat oleh <a class="new" title="Joe Palrang (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Joe_Palrang&amp;action=edit&amp;redlink=1">Joe Palrang</a>.</p>
<p>Pertemuan proyek berlangsung di sebuah gedung perkantoran <i>Sand Hill Road</i> di <a class="new" title="Menlo Park (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Menlo_Park&amp;action=edit&amp;redlink=1">Menlo Park</a>. Sekitar musim panas 1992 proyek ini ditutup dengan menghasilkan sebuah program <i>Java Oak</i> pertama, yang ditujukan sebagai pengendali sebuah peralatan dengan teknologi layar sentuh (<i>touch screen</i>), seperti pada PDA sekarang ini. Teknologi baru ini dinamai &#8220;*7&#8243; (<i>Star Seven</i>).</p>
<p>Setelah era <i>Star Seven</i> selesai, sebuah anak perusahaan <a class="mw-redirect" title="Tv kabel" href="http://id.wikipedia.org/wiki/Tv_kabel">Tv kabel</a> tertarik ditambah beberapa orang dari proyek The Green Project. Mereka memusatkan kegiatannya pada sebuah ruangan kantor di 100 Hamilton Avenue, <a class="mw-disambig" title="Palo Alto" href="http://id.wikipedia.org/wiki/Palo_Alto">Palo Alto</a>.</p>
<p>Perusahaan baru ini bertambah maju: jumlah karyawan meningkat dalam waktu singkat dari 13 menjadi 70 orang. Pada rentang waktu ini juga ditetapkan pemakaian <a title="Internet" href="http://id.wikipedia.org/wiki/Internet">Internet</a> sebagai medium yang menjembatani kerja dan ide di antara mereka. Pada awal tahun 1990-an, Internet masih merupakan rintisan, yang dipakai hanya di kalangan <a class="new" title="Akademisi (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Akademisi&amp;action=edit&amp;redlink=1">akademisi</a> dan <a title="Militer" href="http://id.wikipedia.org/wiki/Militer">militer</a>.</p>
<p>Mereka menjadikan <a class="mw-redirect" title="Perambah" href="http://id.wikipedia.org/wiki/Perambah">perambah</a> (<i>browser</i>) <a class="new" title="Mosaic (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Mosaic&amp;action=edit&amp;redlink=1">Mosaic</a> sebagai landasan awal untuk membuat perambah Java pertama yang dinamai Web Runner, terinsipirasi dari film 1980-an, Blade Runner. Pada perkembangan rilis pertama, Web Runner berganti nama menjadi Hot Java.</p>
<p>Pada sekitar bulan <a title="Maret" href="http://id.wikipedia.org/wiki/Maret">Maret</a> <a title="1995" href="http://id.wikipedia.org/wiki/1995">1995</a>, untuk pertama kali kode sumber Java versi 1.0a2 dibuka. Kesuksesan mereka diikuti dengan untuk pemberitaan pertama kali pada surat kabar <i>San Jose Mercury News</i> pada tanggal <a title="23 Mei" href="http://id.wikipedia.org/wiki/23_Mei">23 Mei</a> <a title="1995" href="http://id.wikipedia.org/wiki/1995">1995</a>.</p>
<p>Sayang terjadi perpecahan di antara mereka suatu hari pada pukul 04.00 di sebuah ruangan hotel <a class="new" title="Sheraton Palace (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Sheraton_Palace&amp;action=edit&amp;redlink=1">Sheraton Palace</a>. Tiga dari pimpinan utama proyek, <a title="Eric Schmidt" href="http://id.wikipedia.org/wiki/Eric_Schmidt">Eric Schmidt</a>dan <a class="new" title="George Paolini (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=George_Paolini&amp;action=edit&amp;redlink=1">George Paolini</a> dari <a title="Sun Microsystems" href="http://id.wikipedia.org/wiki/Sun_Microsystems">Sun Microsystems</a> bersama <a class="new" title="Marc Andreessen (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Marc_Andreessen&amp;action=edit&amp;redlink=1">Marc Andreessen</a>, membentuk <a title="Netscape" href="http://id.wikipedia.org/wiki/Netscape">Netscape</a>.</p>
<p>Nama Oak, diambil dari pohon oak yang tumbuh di depan jendela ruangan kerja &#8220;Bapak Java&#8221;, <a title="James Gosling" href="http://id.wikipedia.org/wiki/James_Gosling">James Gosling</a>. Nama Oak ini tidak dipakai untuk versi release Java karena sebuah perangkat lunak lain sudah terdaftar dengan merek dagang tersebut, sehingga diambil nama penggantinya menjadi &#8220;Java&#8221;. Nama ini diambil dari kopi murni yang digiling langsung dari biji (kopi tubruk) kesukaan Gosling. Konon kopi ini berasal dari Pulau <a title="Jawa" href="http://id.wikipedia.org/wiki/Jawa">Jawa</a>. Jadi nama bahasa pemrograman Java tidak lain berasal dari kata Jawa (bahasa Inggris untuk Jawa adalah Java).</p>
<h2><span id="Versi_awal" class="mw-headline">Versi awal</span></h2>
<p>Versi awal Java pada tahun <a title="1996" href="http://id.wikipedia.org/wiki/1996">1996</a> sudah merupakan versi release sehingga dinamakan Java Versi 1.0. Java versi ini menyertakan banyak paket standar awal yang terus dikembangkan pada versi selanjutnya:</p>
<ul>
<li><code>java.lang</code>: Peruntukan kelas elemen-elemen dasar.</li>
<li><code>java.io</code>: Peruntukan kelas <i>input</i> dan <i>output</i>, termasuk penggunaan berkas.</li>
<li><code>java.util</code>: Peruntukan kelas pelengkap seperti kelas struktur data dan kelas kelas penanggalan.</li>
<li><code>java.net</code>: Peruntukan kelas <a class="mw-redirect" title="TCP/IP" href="http://id.wikipedia.org/wiki/TCP/IP">TCP/IP</a>, yang memungkinkan berkomunikasi dengan komputer lain menggunakan jaringan TCP/IP.</li>
<li><code>java.awt</code>: Kelas dasar untuk aplikasi antarmuka dengan pengguna (GUI)</li>
<li><code>java.applet</code>: Kelas dasar aplikasi antar muka untuk diterapkan pada penjelajah web.</li>
</ul>
<h2><span id="Kelebihan" class="mw-headline">Kelebihan</span></h2>
<ul>
<li><i><b>Multiplatform</b></i>. Kelebihan utama dari Java ialah dapat dijalankan di beberapa <i>platform</i> / <a title="Sistem operasi" href="http://id.wikipedia.org/wiki/Sistem_operasi">sistem operasi</a> komputer, sesuai dengan prinsip <i>tulis sekali, jalankan di mana saja</i>. Dengan kelebihan ini pemrogram cukup menulis sebuah program Java dan dikompilasi (diubah, dari bahasa yang dimengerti manusia menjadi bahasa mesin / <i>bytecode</i>) sekali lalu hasilnya dapat dijalankan di atas beberapa platform tanpa perubahan. Kelebihan ini memungkinkan sebuah program berbasis java dikerjakan diatas operating system Linux tetapi dijalankan dengan baik di atas Microsoft Windows. Platform yang didukung sampai saat ini adalah<a title="Microsoft Windows" href="http://id.wikipedia.org/wiki/Microsoft_Windows">Microsoft Windows</a>, <a title="Linux" href="http://id.wikipedia.org/wiki/Linux">Linux</a>, <a title="Mac OS" href="http://id.wikipedia.org/wiki/Mac_OS">Mac OS</a> dan <a class="mw-redirect" title="Sun Solaris" href="http://id.wikipedia.org/wiki/Sun_Solaris">Sun Solaris</a>. Penyebabnya adalah setiap sistem operasi menggunakan programnya sendiri-sendiri (yang dapat diunduh dari situs Java) untuk meninterpretasikan <i>bytecode</i> tersebut.</li>
</ul>
<ul>
<li><b><a class="mw-redirect" title="OOP" href="http://id.wikipedia.org/wiki/OOP">OOP</a></b> (<i>Object Oriented Programming</i> &#8211; Pemrogram Berorientasi Objek)</li>
<li><b>Perpustakaan Kelas Yang Lengkap</b>, Java terkenal dengan kelengkapan <i>library</i>/<a class="new" title="Perpustakaan (pemrograman) (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Perpustakaan_(pemrograman)&amp;action=edit&amp;redlink=1">perpustakaan</a> (kumpulan program program yang disertakan dalam pemrograman java) yang sangat memudahkan dalam penggunaan oleh para pemrogram untuk membangun aplikasinya. Kelengkapan perpustakaan ini ditambah dengan keberadaan komunitas Java yang besar yang terus menerus membuat perpustakaan-perpustakaan baru untuk melingkupi seluruh kebutuhan pembangunan aplikasi.</li>
</ul>
<ul>
<li><b>Bergaya C++</b>, memiliki sintaks seperti bahasa pemrograman <a title="C++" href="http://id.wikipedia.org/wiki/C%2B%2B">C++</a> sehingga menarik banyak pemrogram C++ untuk pindah ke Java. Saat ini pengguna Java sangat banyak, sebagian besar adalah pemrogram C++ yang pindah ke Java. Universitas-universitas di <a title="Amerika Serikat" href="http://id.wikipedia.org/wiki/Amerika_Serikat">Amerika Serikat</a> juga mulai berpindah dengan mengajarkan Java kepada murid-murid yang baru karena lebih mudah dipahami oleh murid dan dapat berguna juga bagi mereka yang bukan mengambil jurusan komputer.</li>
</ul>
<ul>
<li><b><a class="mw-redirect" title="Pengumpulan sampah" href="http://id.wikipedia.org/wiki/Pengumpulan_sampah">Pengumpulan sampah</a></b> otomatis, memiliki fasilitas pengaturan penggunaan memori sehingga para pemrogram tidak perlu melakukan pengaturan memori secara langsung (seperti halnya dalam bahasa C++ yang dipakai secara luas).</li>
</ul>
<h2><span id="Kekurangan" class="mw-headline">Kekurangan</span></h2>
<ul>
<li><b>Tulis sekali, jalankan di mana saja</b> &#8211; Masih ada beberapa hal yang tidak kompatibel antara <i><a title="Platform" href="http://id.wikipedia.org/wiki/Platform">platform</a></i> satu dengan <i>platform</i> lain. Untuk <a class="new" title="J2SE (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=J2SE&amp;action=edit&amp;redlink=1">J2SE</a>, misalnya <i>SWT-AWT bridge</i> yang sampai sekarang tidak berfungsi pada <a class="mw-redirect" title="Mac OS X" href="http://id.wikipedia.org/wiki/Mac_OS_X">Mac OS X</a>.</li>
</ul>
<ul>
<li><b>Mudah didekompilasi</b>. Dekompilasi adalah proses membalikkan dari kode jadi menjadi kode sumber. Ini dimungkinkan karena kode jadi Java merupakan<i>bytecode</i> yang menyimpan banyak atribut bahasa tingkat tinggi, seperti nama-nama kelas, metode, dan tipe data. Hal yang sama juga terjadi pada <a class="new" title="Microsoft .NET Platform (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Microsoft_.NET_Platform&amp;action=edit&amp;redlink=1">Microsoft .NET Platform</a>. Dengan demikian, algoritma yang digunakan program akan lebih sulit disembunyikan dan mudah dibajak/di<i>reverse-engineer</i>.</li>
</ul>
<ul>
<li><b>Penggunaan memori yang banyak</b>. Penggunaan memori untuk program berbasis Java jauh lebih besar daripada bahasa tingkat tinggi generasi sebelumnya seperti C/<a title="C++" href="http://id.wikipedia.org/wiki/C%2B%2B">C++</a> dan <a class="mw-disambig" title="Pascal" href="http://id.wikipedia.org/wiki/Pascal">Pascal</a> (lebih spesifik lagi, <a class="mw-disambig" title="Delphi" href="http://id.wikipedia.org/wiki/Delphi">Delphi</a> dan <a class="new" title="Object Pascal (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Object_Pascal&amp;action=edit&amp;redlink=1">Object Pascal</a>). Biasanya ini bukan merupakan masalah bagi pihak yang menggunakan teknologi terbaru (karena trend memori terpasang makin murah), tetapi menjadi masalah bagi mereka yang masih harus berkutat dengan mesin komputer berumur lebih dari 4 tahun.</li>
</ul>

<h2><span id="Tahap_kompilasi" class="mw-headline">Tahap kompilasi</span></h2>
<ol>
<li>Tulis / Ubah. Pemrogram menulis program dan menyimpannya di media dalam bentuk berkas &#8216;.java&#8217;.</li>
<li>Kompilasi. Pengkompilasi membentuk <i>bytecodes</i> dari program menjadi bentuk berkas &#8216;.class&#8217;.</li>
<li>Muat. Pemuat kelas memuat <i>bytecodes</i> ke memori.</li>
<li>Verifikasi. Peng-verifikasi memastikan <i>bytecodes</i> tidak mengganggu sistem keamanan Java.</li>
<li>Jalankan. Penerjemah menerjemahkan <i>bytecodes</i> ke bahasa mesin.tidak bisa di pakai‘’</li>
</ol>
<h2><span id="Integrated_Development_Environment" class="mw-headline">Integrated Development Environment</span></h2>
<p>Banyak pihak telah membuat <a class="mw-disambig" title="IDE" href="http://id.wikipedia.org/wiki/IDE">IDE</a> (<i>Integrated Development Environment</i> &#8211; Lingkungan Pengembangan Terintegrasi) untuk Java. Yang populer saat ini (Juli 2006) antara lain:</p>
<ul>
<li><a class="external text" href="http://www.drjava.org/" rel="nofollow">Dr. Java, program gratis yang dikembangkan oleh</a> <a class="new" title="Universitas Rice (halaman belum tersedia)" href="http://id.wikipedia.org/w/index.php?title=Universitas_Rice&amp;action=edit&amp;redlink=1">Universitas Rice</a>, <a title="Amerika Serikat" href="http://id.wikipedia.org/wiki/Amerika_Serikat">Amerika Serikat</a></li>
<li><a class="external text" href="http://www.bluej.org/" rel="nofollow">BlueJ, program gratis yang dikembangkan oleh</a> <a title="Universitas Monash" href="http://id.wikipedia.org/wiki/Universitas_Monash">Universitas Monash</a>, <a title="Australia" href="http://id.wikipedia.org/wiki/Australia">Australia</a></li>
<li><a class="external text" href="http://www.netbeans.org/products/ide/" rel="nofollow">NetBeans</a> (open source- <a class="external text" href="http://www.netbeans.org/about/legal/license.html" rel="nofollow">Common Development and Distribution License (CDDL)</a>)</li>
</ul>
<p>NetBeans disponsori Sun Microsystems, dan versi terkininya memilki Matisse, sebuah GUI Editor yang menurut pendapat umum merupakan yang terbaik.<sup class="noprint Inline-Template"><span title="Kalimat yang diikuti tag ini membutuhkan rujukan.">[<i><a title="Wikipedia:Kutip sumber tulisan" href="http://id.wikipedia.org/wiki/Wikipedia:Kutip_sumber_tulisan">butuh rujukan</a></i>]</span></sup></p>
<ul>
<li><a class="external text" href="http://www.eclipse.org/jdt" rel="nofollow">Eclipse JDT</a> (open source- <a class="external text" href="http://www.eclipse.org/org/documents/epl-v10.php" rel="nofollow">Eclipse Public License</a>)</li>
</ul>
<p><a title="Eclipse (perangkat lunak)" href="http://id.wikipedia.org/wiki/Eclipse_(perangkat_lunak)">Eclipse</a> dibuat dari kerja sama antara perusahaan-perusahaan anggota &#8216;Eclipse Foundation&#8217; (beserta individu-individu lain). Banyak nama besar yang ikut dalam &#8216;Eclipse Foundation&#8217;, termasuk IBM, BEA, Intel, Nokia, Borland. Eclipse bersaing langsung dengan Netbeans IDE. Plugin tambahan pada Eclipse jauh lebih banyak dan bervariasi dibandingkan IDE lainnya.</p>
<ul>
<li><a class="external text" href="http://www.jetbrains.com/idea/" rel="nofollow">IntelliJ IDEA</a> (commercial, free 30-day trial)</li>
<li><a class="external text" href="http://www.oracle.com/tools/jdev_home.html" rel="nofollow">Oracle JDeveloper</a> (free)</li>
<li><a class="external text" href="http://www.jcreator.com/" rel="nofollow">Xinox JCreator</a> (ada versi berbayar maupun free)</li>
</ul>
<p>JCreator ditulis dalam C/C++ sehingga lebih cepat (dan menggunakan memori lebih sedikit) dari kebanyakan.</p>
